## 宠康记（PetHealthNote）架构设计（V1.0）

> 更新时间：2025-11。当前版本对应 1.0 正式交付的能力（登录、宠物&健康档案、疫苗/提醒、遛弯记录、收藏点等），后续演进将在下一阶段另行补充。

### 1. 整体架构概览

- **前端**：微信小程序（`frontend-miniapp/`），包含：首页（宠物列表）、宠物详情、健康/医疗/疫苗记录维护、提醒列表、遛弯记录、常用地点收藏、帮助&关于等页面。
- **后端**：基于 Spring Boot Cloud 的微服务 / 单体演进架构（当前阶段以单应用为主，预留拆分能力）
  - `pet-health-note`：统一提供用户、宠物、健康记录、医疗记录、疫苗记录、提醒、遛弯、收藏、天气等 API
  - 预留：日后可拆分为用户中心、宠物健康服务、统计与报表服务等
- **数据层**：
  - MySQL：核心业务数据
  - Redis：缓存（后续阶段引入）
- **存储**：
  - 对象存储（MinIO / 阿里云 OSS 等）：用于存放宠物头像、就诊单据图片等（后续阶段实现）

### 2. 后端模块划分

在 `com.pethealthnote` 包下按领域划分模块，当前结构与实现一致：

- `common`：通用组件与基础配置
  - 全局异常处理
  - 统一返回结果封装
  - 通用常量、枚举
- `user`：用户与账号
  - 用户基本信息
  - 微信登录与身份标识绑定（openId/unionId）
- `auth`：登录认证
  - 微信 `code` 登录接口
  - token 生成与前端持久化约定
- `pet`：宠物信息
  - 宠物档案（基本信息、品种、生日、性别等）
- `health`：日常健康记录（体重、体温、饮食、排泄、备注）
- `medical`：医疗记录（就诊信息、费用、处方）
- `vaccine`：疫苗与驱虫记录
- `reminder`：提醒与计划（疫苗 / 驱虫 / 体检 / 其他）
- `favorite`：常用地点/医生收藏，供提醒或遛弯引用
- `stats`：统计与报表（后续阶段逐步补充）

### 3. 技术选型（后端）

- Spring Boot 2.7.x
- Spring Cloud 2021.x（按需增加注册中心、配置中心、网关等）
- MyBatis Plus
- MySQL 8.x
- Lombok

### 4. 调用关系（简化）

微信小程序前端 ⟶ Nginx（SSL/静态） ⟶ `pet-health-note` 服务 ⟶ MySQL / （预留）Redis / （预留）对象存储。

### 5. 配置管理

- 当前阶段使用 `application.yml` + profile（dev/prod）管理配置。
- 后续可引入 Spring Cloud Config 或 Nacos 统一配置中心。

### 6. 后续演进方向（占位）

- 接入微信 `code2Session`，将当前模拟登录替换为正式鉴权，并扩展 token 鉴权链路
- 引入网关、统一鉴权与限流
- 引入消息队列实现异步通知与事件驱动
- 引入统一监控与链路追踪（Prometheus/Zipkin 等）
